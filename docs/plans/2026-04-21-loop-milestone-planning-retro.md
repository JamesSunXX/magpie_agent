# Loop 里程碑实施流程补充记录

## 背景

这次改动的目标是把 `loop` 中“TRD 生成后直接进入实现”的流程，拆成“先列实施里程碑，再按里程碑进入实现”。

原本应该先写计划文档再动代码。本次执行时漏掉了这个步骤，所以本文档是事后补充记录，不作为事前计划冒充。

## 完成标准

- 默认 `loop` 阶段包含 TRD 后的里程碑计划步骤。
- 里程碑计划写入会话目录，后续恢复时路径稳定。
- 实现阶段能读取并带上这份里程碑计划。
- 配置、初始化模板、文档说明和测试同步更新。
- 项目要求的测试、构建、文档检查和边界检查通过。

## 实际改动

- 新增 `milestone_planning` 阶段，默认放在 `trd_generation` 之后、开发准备之前。
- 新增 `milestone-plan.json` 会话产物，用来记录有序实施里程碑。
- 实现阶段提示会读取 `milestone-plan.json`，要求按最早未完成的里程碑推进。
- 更新阶段枚举、配置校验、初始化配置、阶段绑定和文档路由。
- 更新 README 和能力说明，把默认主线从 9 段调整为 10 段。
- 补充主流程、计划器、阶段绑定、配置校验和初始化配置相关测试。

## 验证结果

- `npm run test:run`：通过，1030 个测试全部通过。
- `npm run test:coverage`：通过，触碰到的 loop 文件覆盖率满足要求。
- `npm run build`：通过。
- `npm run lint`：通过。
- `npm run check:docs`：通过。
- `npm run check:boundaries`：通过。
